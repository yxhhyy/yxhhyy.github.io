SQL宽字节注入

宽字节的介绍

  1. GBK 是占两个字节（也就是名叫宽字节，只要字节大于1的都是）
  2. ASCII 占一个字节
  3. PHP中编码为GBK ，函数执行添加的是ASCII编码，mysql 默认字符集是GBK等宽字节字符集

为什么要介绍宽字节？

宽字节的出现个人感觉是因为SQL语句过滤了一些字符，比如 ’ 转义成 /’ 或者 “ 转义成 /“（就是在字符前面加上一个转义字符）

在这种情况下就不容易完成注入，所以出现了宽字节注入，通过宽字节绕过转义字符

宽字节注入原理

宽字节注入是一种SQL注入攻击技术，它利用了某些字符集（如GBK）的特性来绕过对特殊SQL字符（如单引号）的转义处理。在宽字节字符集中，一个字符可能占用两个字节，而在某些情况下，这可以被用来绕过安全过滤。

在PHP中，当使用GBK编码时，如果前一个ASCII码大于128，MySQL会将两个字符视为一个汉字。这意味着，如果攻击者在注入点前插入一个宽字节字符，如 %df，并跟随一个单引号，这个单引号就不会被转义函数（如addslashes()）正确处理。例如，%df' 经过转义后变成 %df\'，在GBK编码下，%df%5c（*%5c* 是反斜杠的URL编码）会被视为一个汉字，从而使得单引号 %27 成功逃逸，导致注入攻击的发生。

过滤的函数

  1. mysql_real_escape_string()

  2. addslashes()

    addslashes() 是 PHP 中的一个内置函数，它的作用是在字符串前添加反斜杠（\）以转义特殊字符，防止它们被解析为特殊的SQL语法。这些特殊字符通常包括单引号（’）、双引号（”）、反斜杠（\）、NULL 字符（NUL）、非打印控制字符以及某些数据库特定的转义字符。例如，如果你有一个用户输入的字符串打算插入到 SQL 查询中，为了安全起见，可以先用 addslashes() 函数处理，以防恶意注入攻击。

绕过原理

  1. 比如使用 %df’：会被PHP当中的 addslashes 函数转义为“ %df\’ ”。
  2. “ \ ”即 url 里面的“ %5c ”, “ ’ ”对应的 url 编码是“%27”，那么也就是说，“ %df’ ”会被转义“ %df%5c%27 ”
  3. 倘若网站的字符集是GBK，mysql 使用的编码也是GBK的话，就会认为“ %df%5c%27 ”是一个宽字节。
  4. “ %df%5c ”会结合（因为宽字节是占两个字节），也就是“縗”（两个字符就会被视为一个汉字） 。后面就有一个“ ’ ”。这么一来，' 就成功逃逸，达到了注入的目的

使用其他宽字节

  1. 不仅仅只是使用 %df’ 进行宽字节绕过也可以使用其他的宽字节，只要满足字符串编码的要求
  2. 常见使用的宽字节就是 %df ,其实 当我们输入第一个ascill大于128就可以，转换是将其转换成16进制，例如：129转换0x81,然后在前面加上%就是%81
  3. GBK首字节对应 **0x81-0xef(129-239)**，尾字节对应 0x40-0x7e(64-126)【除了0x80（128）】
  4. 比如一些 %df’ %81’ %82’ %de’ 等等（只要满足上面的要求就可以）

通过 SQLmap 进行演示

sqlmap.py -u "url" --batch --tamper=unmagicquotes.py --current-db // 查询数据库
unmagicquotes.py 是专门进行宽字节绕过脚本

sqlmap.py -u "url" --batch --tamper=unmagicquotes.py -D security --tables // 数据表

sqlmap.py -u "url" --batch --tamper=unmagicquotes.py -D security -T users --columns // 查询列名

sqlmap.py -u "url" --batch --tamper=unmagicquotes.py -D security -T users -C "username,password" --dump // 详细数据
